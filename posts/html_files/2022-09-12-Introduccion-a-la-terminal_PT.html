Formato do post¶

Para não ter que ficar colocando imagens do console em cada ação que faço, criei a seguinte função que recebe o comando do terminal que queremos executar e devolve a saída que o terminal nos daria. Às vezes usarei esta função, e outras usarei ! antes de cada comando, que em notebooks quer dizer que você vai executar um comando do terminal.

import subprocess
import os

last_directory = ''

def terminal(command, max_lines_output=None):
    global last_directory
    debug = False
    str = command.split()

    # Check if there are " or ' characters
    for i in range(len(str)):
        if debug:   print(f"i = {i}, str[i] = {str[i]}")
        if len(str[i]) > 0:
            if str[i][0] == '"' or str[i][0] == "'":
                for j in range(i+1,len(str)):
                    if debug:   print(f"\t j = {j}, str[j] = {str[j]}")
                    if str[j][-1] == '"' or str[j][-1] == "'":
                        for k in range(i+1,j+1):
                            if debug:   print(f"\t\t k = {k}, str[i] = {str[i]}, str[k] = {str[k]}")
                            str[i] = str[i] + " " + str[k]
                            if debug:   print(f"\t\t k = {k}, str[i] = {str[i]}, str[k] = {str[k]}")
                        str[j:] = [""]
                        str[i] = str[i].replace('"','')
    
    # Remove empty strings
    str = [x for x in str if x != ""]
    if debug:
        print(str)
        return
    if str[0] == "cd":
        last_dir = os.getcwd()
        if len(str) == 1:
            os.chdir('/home/wallabot')
        else:
            if str[1] == "-":
                os.chdir(last_directory)
            else:
                os.chdir(str[1])
        last_directory = last_dir
    else:
        result = subprocess.run(str, stdout=subprocess.PIPE).stdout.decode('utf-8')
        if max_lines_output is not None:
            result_split = result.split('\n')
            print('\n'.join(result_split[:max_lines_output]))
            print("\t ...")
            print('\n'.join(result_split[-5:]))
        else:
            print(result)

Sincronizar arquivos mediante rsync¶






Até agora vimos como copiar, mover e excluir arquivos, mas suponhamos que temos uma pasta e copiamos esses arquivos para outra. Agora suponhamos que modificamos um arquivo da primeira pasta e queremos que a segunda tenha as mudanças. Temos duas opções, voltar a copiar todos os arquivos, ou fazer uma sincronização usando rsync






Primeiro vamos criar uma nova pasta na qual criaremos vários arquivos








!mkdir sourcefolder
!touch sourcefolder/file1 sourcefolder/file2 sourcefolder/file3









Agora criamos uma segunda pasta que é a que vamos sincronizar com a primeira








!mkdir syncfolder











!echo "ls sourcefolder:" && ls sourcefolder && echo "ls syncfolder:" && ls syncfolder













ls sourcefolder:
file1  file2  file3
ls syncfolder:













Sincronizamos as duas pastas com rsync, a primeira vez apenas copiará os arquivos da primeira pasta para a segunda. Para fazer isso, além disso, devemos adicionar a flag -r (recursive)








!rsync -r sourcefolder/ syncfolder/











!echo "ls sourcefolder:" && ls sourcefolder && echo "ls syncfolder:" && ls syncfolder













ls sourcefolder:
file1  file2  file3
ls syncfolder:
file1  file2  file3













Se agora eu criar um novo arquivo em sourcefolder e sincronizar novamente, apenas esse arquivo será copiado para o syncfolder. Para ver que apenas um arquivo será copiado, podemos usar o flag -v (verbose)








!touch sourcefolder/file4











!rsync -r -v sourcefolder/ syncfolder/













sending incremental file list
file1
file2
file3
file4

sent 269 bytes  received 92 bytes  722.00 bytes/sec
total size is 0  speedup is 0.00













Mas parece que copiou todos os arquivos, então para que isso não aconteça e copie apenas os que foram modificados, é preciso usar a flag -u








!touch sourcefolder/file5











!rsync -r -v -u sourcefolder/ syncfolder/













sending incremental file list
file5

sent 165 bytes  received 35 bytes  400.00 bytes/sec
total size is 0  speedup is 0.00















!echo "ls sourcefolder:" && ls sourcefolder && echo "ls syncfolder:" && ls syncfolder













ls sourcefolder:
file1  file2  file3  file4  file5
ls syncfolder:
file1  file2  file3  file4  file5













E o que acontece se eu criar um novo arquivo em syncfolder?








!touch syncfolder/file6











!rsync -r -v -u sourcefolder/ syncfolder/

Não o sincroniza, então é importante ter isso em mente






Algumas flags importantes a serem levadas em consideração são:


    

  • -a: Este flag é um atalho para várias opções, incluindo -r (recursivo), -l (copiar links simbólicos), -p (manter permissões), -t (manter o horário de modificação) e -g (manter grupo). Esta opção é útil para fazer uma cópia exata de um diretório, incluindo todas as suas subpastas e arquivos. * -v: Este flag ativa a saída detalhada, que mostra os arquivos que estão sendo copiados e o progresso da operação. * -r: Esta flag é utilizada para copiar de forma recursiva, o que significa que copia todas as subpastas e arquivos dentro de um diretório. * -u: Esta flag é utilizada para copiar apenas os arquivos novos ou modificados. Se um arquivo já existe no destino e é mais recente que o arquivo de origem, não é copiado. * -n: Esta flag é usada para fazer um teste de cópia, o que significa que não são efetuadas alterações no destino.* --exclude: Esta opção é usada para excluir arquivos ou pastas específicos da operação de cópia. Você pode especificar vários arquivos ou pastas para excluir utilizando esta opção várias vezes. * -z: Este flag é utilizado para comprimir os dados durante a transferência, o que reduz a largura de banda utilizada e acelera a velocidade de transferência.* -h: esta flag é usada para mostrar a informação em um formato mais legível, especialmente quando se trabalha com grandes quantidades de dados ou tamanhos de arquivos grandes.

Os seis tipos de repositórios do Ubuntu são: 1. Principal
O repositório Main está ativado por padrão e contém apenas software livre e de código aberto ou FOSS pela sua sigla em inglês (Free and Open-Source Software).
 2. Universo
Como Main, Universe também oferece FOSS. A diferença é que neste repositório não é o Ubuntu quem garante atualizações de segurança regulares, mas sim a comunidade que se encarrega de seu suporte. Vem ativado por padrão, mas nem sempre. Alguns sistemas operacionais o têm desativado por padrão e podemos ter que ativá-lo se estivermos executando uma Live Session. Se não o tivermos adicionado, podemos fazê-lo com este comando:


```sudo add-apt-repository universe```
## O que encontramos no «Universe»? Eu diria que a maioria do software que vale a pena, entre os quais temos o VLC ou o OpenShot.
3. Multiverso
A partir daqui vêm os repositórios do Ubuntu com menos liberdade. `Multiverse` contém software que já não é `FOSS` e o Ubuntu não pode ativar este repositório por padrão devido a problemas legais e de licenças. Por outro lado, também não pode fornecer patches e atualizações. Com isso em mente, temos que considerar se o adicionamos ou não, algo que podemos fazer com este comando:
``` bash
```sudo add-apt-repository multiverse
4. Restrito
Nos repositórios do Ubuntu, podemos encontrar software livre e de código aberto, mas isso não é possível quando se trata de algo relacionado com o hardware. Nos repositórios `Restricted`, encontraremos drivers, como os de placas gráficas, painéis tácteis ou placas de rede.
``` bash



sudo add-apt-repository restricted
5. Parceiro
Este repositório contém software proprietário compilado pelo Ubuntu de seus parceiros.
6. Repositórios de terceiros do Ubuntu
Por fim, temos os repositórios de terceiros. O Ubuntu tenta oferecer sempre a melhor experiência do usuário e esse é um dos motivos pelo qual rejeita certo software. Também há desenvolvedores que preferem ter um controle total sobre o que oferecem e por esse motivo criam seus próprios repositórios.

Atalhos de teclado¶






A seguir são mostrados alguns atalhos de teclado úteis no uso do terminal * Ctrl+a: leva o cursor ao início da linha de comandos. * Ctrl+e: leva o cursor para o final da linha de comando. * Ctrl+l: limpa o terminal, similar ao que faz o comando clear.* Ctrl+u: limpa desde a posição do cursor até o início da linha.
